# my-skills

내가 AI 코딩 도구에게 반복해서 시키던 일을 재사용 가능한 스킬로 만들어 모아두는 저장소입니다.

## wrap-up — 작업 끝나면 알아서 커밋 정리

### 어떤 불편이 있었나

기능 하나를 고칠 때마다 매번 이 세 가지를 반복해야 했습니다.

1. "이거 뭐가 바뀌었지" — 상태를 다시 설명해야 함
2. "커밋 메시지는 이런 톤으로" — 이 프로젝트의 스타일을 매번 다시 알려줘야 함
3. "이제 커밋해줘" — 매번 별도로 명시적으로 요청해야 함

같은 대화 안에서도 기능 하나 끝날 때마다 "커밋"이라고 따로 타이핑해야 했고,
새 대화나 새 프로젝트에서는 저 세 단계를 처음부터 다시 설명해야 했습니다.

### 스킬이 어떻게 없앴나

`.claude/skills/wrap-up/SKILL.md` 하나만 있으면 "커밋해줘" / "wrap this up" 한마디로:

- 실제 `git status`/`git diff`를 직접 확인하고 (추측 없이)
- 이 저장소의 기존 커밋 스타일(`git log`)을 스스로 파악해서 맞추고
- 관련 있는 파일만 골라 스테이징하고
- "무엇을"이 아니라 "왜"에 집중한 메시지로 커밋합니다

push는 절대 자동으로 하지 않습니다 — push는 항상 별도의 명시적 요청이 있을 때만 합니다.

## 설치

이 저장소의 `.claude/skills/wrap-up/`을 그대로 프로젝트의 `.claude/skills/`
(또는 사용자 전역 `~/.claude/skills/`)에 복사하면 Claude Code에서 바로 인식합니다.

Claude Code가 아닌 다른 도구를 쓴다면, 그 도구의 스킬/커스텀 명령 위치
(예: Cursor의 `.cursor/rules`, 그 외 도구의 커스텀 프롬프트 폴더)에
`SKILL.md`의 frontmatter를 뺀 본문 내용을 붙여 넣으면 동일하게 동작합니다.

## 사용법

아무 저장소에서나:

```
커밋해줘
```

또는

```
wrap this up
```

라고만 말하면 됩니다. 별도 설명 없이 바로 동작해야 정상입니다.

# my-skills

내가 AI 코딩 도구에게 반복해서 시키던 일을 재사용 가능한 스킬로 만들어 모아두는 저장소입니다.

## verify-locally — 고쳤다고 말하기 전에 실제로 확인하기

### 어떤 불편이 있었나

로컬 서버(FastAPI 앱)를 고치는 작업을 반복하면서, 매번 같은 문제를 겪었습니다.

1. 코드를 고치고 "이론상 맞다"는 이유로 바로 "고쳤습니다"라고 보고했다가, 실제로는
   diff만 봐서는 안 보이던 버그(카테고리 매칭 오류, 시간대 변환 오류, 필드 기본값 누락 등)가
   실제 호출에서만 드러났습니다.
2. 이미 다른 프로세스가 떠 있는 포트에 그대로 테스트했다가 충돌이 나거나, 실사용 데이터베이스에
   테스트 데이터가 섞여 들어갔습니다.
3. 한글이 포함된 요청을 커맨드라인에 직접 넣었다가 셸(Git Bash)이 인코딩을 깨뜨려서,
   진짜 버그가 아닌 "가짜 에러"를 붙잡고 시간을 낭비했습니다.

이 세 가지를 매번 새로 설명하거나 매번 다시 겪는 대신, 한 번 정리해서 스킬로 남겼습니다.

### 스킬이 어떻게 없앴나

`.claude/skills/verify-locally/SKILL.md` 하나면, 코드를 고친 뒤 AI가 스스로:

- 기존 프로세스와 충돌하지 않는 **스크래치 포트**를 골라 서버를 띄우고
- 이번에 고친 부분과 **직접 관련된 엔드포인트만** 실제로 호출해서 확인하고
- 한글 등 비ASCII 요청 바디는 **파일로 써서 전달**해 인코딩 문제를 피하고
- 검증 중 만든 **테스트 데이터/로그/프로세스를 정리**하고
- 이 과정을 거친 뒤에만 "확인했다"고 보고하고, 못 했으면 못 했다고 명시합니다.

## 설치

이 저장소의 `.claude/skills/verify-locally/`를 그대로 프로젝트의 `.claude/skills/`
(또는 사용자 전역 `~/.claude/skills/`)에 복사하면 Claude Code에서 바로 인식합니다.

Claude Code가 아닌 다른 도구를 쓴다면, 그 도구의 스킬/커스텀 명령 위치
(예: Cursor의 `.cursor/rules`, 그 외 도구의 커스텀 프롬프트 폴더)에
`SKILL.md`의 frontmatter를 뺀 본문 내용을 붙여 넣으면 동일하게 동작합니다.

## 사용법

로컬 서버가 있는 어떤 프로젝트에서든, 기능을 고친 뒤:

```
확인해줘
```

또는

```
verify this works
```

라고만 말하면 됩니다. 별도 설명 없이 바로 동작해야 정상입니다.

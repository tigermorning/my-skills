# my-skills

내가 AI 코딩 도구에게 반복해서 시키던 일을 재사용 가능한 스킬로 만들어 모아두는 저장소입니다.
여기 있는 스킬은 **어느 프로젝트에서나 그대로 쓸 수 있는 범용 스킬**만 다룹니다.
프로젝트에 강하게 종속된 스킬은 이 저장소에 두지 않고, 각 프로젝트 안에 남겨둔 뒤
아래 "프로젝트 전용 스킬 카탈로그"에서 위치만 기록합니다 (필요할 때 찾아서 참고 또는 변형).

## 설치 (2026-07-23부터: 사용자 전역 설치가 기본)

이 저장소의 스킬들은 `~/.claude/skills/omc-learned/`에 이미 설치되어 있어서,
**Claude Code를 쓰는 한 새 프로젝트를 만들 때 더 이상 복사가 필요 없습니다** — 자동으로 인식됩니다.

다른 컴퓨터에 새로 세팅하거나, 특정 프로젝트에만 국한해서 쓰고 싶다면:

```bash
# 사용자 전역 (모든 프로젝트에 자동 적용)
cp -r .claude/skills/<스킬명> ~/.claude/skills/omc-learned/<스킬명>

# 또는 프로젝트 한정
cp -r .claude/skills/<스킬명> /path/to/project/.claude/skills/<스킬명>
```

Claude Code가 아닌 다른 도구를 쓴다면, 그 도구의 스킬/커스텀 명령 위치
(예: Cursor의 `.cursor/rules`, 그 외 도구의 커스텀 프롬프트 폴더)에
`SKILL.md`의 frontmatter를 뺀 본문 내용을 붙여 넣으면 동일하게 동작합니다.

## 범용 스킬 카탈로그

### non-ascii-via-file — 한글을 명령어에 직접 넣지 않기

Windows Git Bash 등에서 터미널 명령어(curl, git commit -m, python -c 등)에 한글을
직접 넣으면 실제 프로그램에 전달되기 전에 인코딩이 깨집니다. 이게 겉으로는 진짜
버그처럼 보여서("요청 형식이 잘못됐다") 멀쩡한 코드를 의심하며 시간을 낭비하기
쉽습니다. 이 스킬은 비ASCII 텍스트를 파일에 먼저 써두고 명령어가 그 파일을
참조하게 만듭니다. 서버가 있는 프로젝트에 국한되지 않고, 한글이 포함된 값을 셸
명령어로 넘기는 모든 상황에 적용됩니다.

### verify-then-code — 예외 있는 규칙은 코드보다 정답표 먼저

`korean-subtitle-corrector`에서 "규칙이 이럴 것"이라는 가정으로 문법 로직을 먼저
코딩했다가 같은 오교정 실수를 두 번 반복한 뒤 확립된 절차를 일반화했습니다.
예외가 있을 수 있는 규칙(문법, 세금/법률 조건, 데이터 분류 기준 등)을 다룰 때,
권위 있는 근거로 "입력 → 정답" 표부터 만들고 코드가 그 표를 따라가게 합니다.
문법 교정기뿐 아니라 권위 있는 근거가 존재하는 모든 규칙 기반 도메인에 적용됩니다.

### real-world-review-cycle — 코드 리딩이 아니라 실사용 검증

파서·교정기·분류기·생성기처럼 입력 공간이 넓은 로직은 코드를 아무리 꼼꼼히
읽어도 실제 입력에서만 드러나는 버그를 놓치기 쉽습니다. 실사용 데이터로 돌려서
사람이 직접 대조하고, 확실한 버그는 즉시 근본 원인을 고치고, 판단이 필요한
애매한 사례는 코드를 건드리지 않고 플래그만 남기는 반복 절차입니다.

### external-search-cache-proxy — 쿼터 있는 외부 검색 API는 캐시+검증 프록시로

`Fridge2Plate`(포크해서 참고 중인 프로젝트)의 YouTube Data API 연동 코드를
읽고 일반화했습니다. 쿼터·비용이 있는 외부 검색/조회 API(YouTube, Google
Maps, OpenAI 등)를 통합할 때, 클라이언트가 직접 부르지 않게 서버 라우트로
감싸고, 캐시 테이블을 먼저 확인하고, 반환된 식별자를 iframe src나 다음
쿼리에 쓰기 전에 형식 정규식으로 검증합니다. (참고: 이 스킬은 직접 겪은
실수가 아니라 남의 코드를 읽고 배운 패턴이라, 실전에서 검증되면서 계속
다듬어질 여지가 있습니다.)

## 프로젝트 전용 스킬 카탈로그 (이 저장소에는 없음, 위치만 기록)

프로젝트 세부사항(파일 경로, PRD 섹션 번호 등)에 강하게 묶여 있어서 그대로
재사용하긴 어렵지만, 비슷한 프로젝트를 새로 시작할 때 참고하거나 변형해서
쓸 수 있는 스킬들입니다.

| 스킬 | 위치 | 용도 |
|---|---|---|
| `session-start-regulation-check` | `korean-subtitle-corrector/.claude/skills/` | 프로젝트가 의존하는 외부 규정(국립국어원 어문 규정)이 세션 시작 시점 기준으로 개정됐는지 확인. **일반화 아이디어**: 외부 스펙/법령/API 버전에 의존하는 프로젝트라면 이 패턴을 "그 스펙 이름"만 바꿔 재사용 가능. |
| `my-todo` | `todo-app/.claude/skills/` | 로컬 Todo 앱을 브라우저 대신 CLI(`cli.py`)로 조작. 이 앱 전용이라 재사용 불가하지만, "로컬 서버 있는 개인 앱을 CLI 스킬로 감싸는" 접근 자체는 다른 개인 프로젝트에도 적용 가능한 패턴. |

## 사용법

한글이 포함된 값을 터미널 명령어로 넘겨야 하거나, 예외 있는 규칙을 코드화하거나,
코드만으로 정확성을 보장하기 어려운 기능을 검증해야 하는 상황이라면, 별도 설명
없이 자연스럽게 발동해야 합니다.
